Prompt

You are a senior Full Stack Software Architect and Engineer.

Your task is to build an MVP for a Tuk-Tuk Sales Management System for an e-mobility company.

The objective is NOT to build a production-ready ERP. Instead, create an MVP that digitizes the company's sales workflow from customer inquiry to vehicle handover.

The application should have a modern, clean, responsive UI that looks professional and enterprise-grade.

Coding Standards
Use functional React components only.
Use custom hooks where appropriate.
Avoid duplicate logic.
Keep components small and reusable.
Use loading states for async operations.
Handle Firebase errors gracefully.
Validate all forms with React Hook Form.
Use optimistic UI updates where appropriate.
Keep the code modular and scalable.
Nice-to-Have Features (If Time Permits)

Real-time notifications using Firebase listeners.
Activity log for major actions (e.g., sale created, payment confirmed, job completed).
Dark mode toggle.
Dashboard filters by date range.
Printable receipts and delivery notes.
Image uploads for vehicles and customer documents using Firebase Storage.
Expected Outcome

Produce a polished MVP that demonstrates the complete business workflow:

Customer registration.
Inquiry creation and tracking.
Inquiry conversion to a sale.
Cash or credit sales processing.
Vehicle reservation and inventory updates.
Parallel Workshop and NTSA processing.
Dispatch readiness validation.
Vehicle handover and sale completion.
Role-based dashboards and permissions.
Real-time synchronization using Firebase Realtime Database.

The application should prioritize clean architecture, maintainability, and an intuitive user experience over implementing every possible feature. The resulting MVP should be suitable for demonstration to stakeholders to validate the business process before expanding into a full enterprise system.
